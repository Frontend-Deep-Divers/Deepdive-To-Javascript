## 비교 연산자

### 동등 비교 연산자 ==

좌항과 우항의 피연산자를 비교할 때 먼저 암묵적 타입 변환을 통해 타입을 일치시킨 후 비교

결과를 예측하기 어려움

### 일치 비교 연산자 ===

좌항과 우항의 피연산자가 타입도 같고 값도 같은 경우에만 true 반환

> NaN <br />
`NaN === NaN // false`
NaN은 자신과 일치하지 않는 유일한 값, `Number.isNaN()` 함수를 통해 검사해야 함
> 

ES6에서 도입된 `Object.is()` 함수로 더 예측 가능한 비교가 가능
