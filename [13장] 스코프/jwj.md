# 13장 스코프

## 13.1 스코프란?

자바스크립트의 스코프는 다른 언어의 스코프와 구별되는 특징이 있다.

**모든 식별자(변수 이름, 함수 이름, 클래스 이름 등)는 자신이 선언된 위치에 의해 다른 코드가 식별자 자신을 참조할 수 있는 유효 범위가 결정된다.**

스코프는 **식별자가 유효한 범위**를 말한다.

자바스크립트 엔진은 스코프를 통해 어떤 변수를 참조할 것인지 결정한다. 스코프는 자바스크립트 엔진이 식별자를 검색할 때 사용하는 규칙이라고도 할 수 있다.

자바스크립트 엔진은 코드를 실행할 때 코드의 문맥을 고려한다. 코드가 실행되는 환경(어디에서 실행되고, 주변에 어떤 코드가 있는지)에 따라 다른 결과가 나올 수 있다.

```js
var x = "global";

function func() {
  var x = "local";
  console.log(x);
}

console.log(x);
```

> 렉시컬 환경(lexical environment): 코드가 실행되는 환경(어디에서 실행되고, 주변에 어떤 코드가 있는지), 실행 컨텍스트를 구성한다.

스코프는 식별자인 변수 이름의 충돌을 방지하여 같은 이름의 변수를 사용할 수 있게 해준다. 다른 스코프에 위치해 있다면, 같은 이름의 식별자를 사용할 수 있다. 즉, **스코프는 네임 스페이스**다.

> `var` 변수는 동일 스코프 내 중복 선언을 허용해 부작용을 발생시킬 수 있다. `let`과 `const` 변수 스코프 내 중복 선언을 허용하지 않는다.

## 13.2 스코프의 종류

소코프는 전역 스코프, 지역 스코프로 나뉜다.

이때 변수는 선언된 위치에 따라 유효 스코프가 결정된다.

- 전역 스코프: 코드의 가장 바깥 영역, 어디서든 참조할 수 있다.
- 지역 스코프: 함수 몸체 내부, 자신의 지역 스코프와 하위 지역 스코프에서 유효하다.

## 13.3 스코프 체인

스코프는 중첩될 수 있다. 스코프는 **함수의 중첩에 의해 계층적인 구조를 갖는다.**

이를 스코프 체인이라 한다.

변수를 참조할 때 자바스크립트 엔진은 **스코프 체인을 따라 변수를 참조하는 코드의 스코프에서 상위 스코프 방향으로 이동하며 선언된 변수를 검색(identifier resolution)한다.**

이는 실제 물리적으로 내부에서 렉시컬 환경의 생성으로 이어진다.

## 13.4 함수 레벨 스코프

💡자바스크립트는 **코드 블록이 아닌 함수에 의해서만 지역 스코프가 생성된다**.

C나 자바 같은 언어에선 함수 몸체를 포함한 모든 코드 블록이 지역 변수를 만드는 블록 레벨 스코프를 사용하지만, 자바스크립트에서 `var`로 선언된 변수는 **오로지 함수의 코드 블록만을 지역 스코프로 인정한다**. (함수 레벨 스코프)

```js
var x = 1;

if (true) {
  var x = 10;
}

console.log(x);
```

`var` 변수는 함수의 코드 블록만을 지역 스코프로 인정하지만, 함수 밖에서 `var`로 선언된 변수는 코드 블록 내에서 선언되었다 하더라도 모두 전역 변수다. 따라서 `if` 문의 `x`는 전역 변수가 된다. `x`는 전역 변수다.

`let`과 `const`는 블록 레벨 스코프를 지원한다.

## 13.5 렉시컬 스코프

```js
var x = 1;

function foo() {
  var x = 10;
  bar();
}

function bar() {
  console.log(x);
}

foo();
bar();
```

위 코드의 실행 결과는 `bar` 함수의 상위 스코프가 무엇인지에 따라 결정될 것이다.

1. 동적 스코프: 함수가 호출된 위치에 따라 상위 스코프를 결정
2. **렉시컬 스코프(정적 스코프)**: 함수가 어디서 정의되었냐에 따라 스코프를 결정

다른 대부분의 언어들과 비슷하게 **자바스크립트는 렉시컬 스코프를 따른다**.

즉, **언제나 자신의 상위 스코프는 자신이 정의된 스코프다.**

따라서 `bar` 함수의 상위 스코프는 전역 스코프이고, 전역 변수 `x` 값인 1을 두 번 출력한다.
