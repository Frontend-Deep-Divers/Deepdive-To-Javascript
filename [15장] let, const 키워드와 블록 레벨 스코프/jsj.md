# 15장 let, const 키워드와 블록 레벨 스코프

## 15.1 `var` 키워드로 선언한 변수의 문제점

- 변수 중복 선언 허용
    - var  키워드로 선언한 변수는 중복 선언이 가능하다..
    - 이로 인해 감지된 에러가 따로 발생하지는 않으나 의도치 않게 값이 변경되는 결과가 나올 수 있다.
- 함수 레벨 스코프
    - var 키워드로 선언한 변수는 오로지 함수의 코드 블록만을 지역 스코프로 인정한다.
    - 함수 외부에서 var 키워드로 선언한 변수는 코드 블록 내에서 선언해도 모두 전역 변수가 된다..
    - 이로 인해 전역 변수가 의도치 않게 중복 선언될 수 있다.
- 변수 호이스팅
    - 변수 선언문 이전에 변수를 참조하게 되어 따로 에러는 발생하지 않지만 가독성을 해치거나 오류가 발생할 수 있다..

## 15.2 `let` 키워드

- ES6에서 새로 추가된 변수 선언 키워드
- 변수 중복 선언 금지
    - let 키워드로 이름이 같은 변수를 중복 선언하면 문법 에러 발생
- 블록 레벨 스코프
    - 모든 코드 블록 (함수, 조건문, 반복문 등등)을 지역 스코프로 인정하는 블록 레벨 스코프를 따른다.
- 변수 호이스팅
    - let 키워드로 선언한 변수는 변수 호이스팅이 발생하지 않는 것 처럼 동작한다.
    - var 키워드로 선언한 변수는 런타임 이전에 자바 스크립트 엔진에 의해 “선언 단계”와 “초기화 단계”가 한번에 진행되어 undefined로 즉시 초기화 된다. 이후 변수 선언문에서 값이 할당된다.
    - 허나 **let 키워드로 선언한 변수는 “선언 단계”와 “초기화 단계”가 분리되어 진행된다.** 런타임 이전에 암묵적으로 선언 단계는 실행되지만 초기화 단계는 변수 선언문에 도달했을 때 진행된다.
    - 이로 인해 **초기화 단계가 실행되기 전에 변수에 접근하면 참조 에러가 발생되며, 스코프의 시작지점(암묵적 선언단계)부터 초기화 단계 시작 지점(변수 선언문)까지 변수를 참조할 수 없다.** 이 구간을 **일시적 사각지대(TDZ)**라고 부른다.
    - 실제로 변수 호이스팅은 발생했으나 TDZ에 의해 참조 에러를 발생시키며 호이스팅이 발생하지 않은 것처럼 작동한다.
- 전역 객체와 let
    - let 키워드로 선언한 전역 변수의 경우 전역 객체의 프로퍼티가 아니게 된다.
    - 따라서 window.x 처럼 (window가 브라우저 전역객체이므로) 접근할 수 없게 된다.
    - 보이지 않는 개념적인 블록 (전역 렉시컬 환경의 선언적 환경 레코드. 23장 참고)에 존재하게 된다.

## 15.3 `const` 키워드

- 상수를 선언하기 위해 사용되지만 반드시 그런건 아니다.
- 선언과 초기화
    - c**onst 키워드로 선언한 변수는 반드시 선언과 동시에 초기화해야 한다.**
    - let 키워드로 선언한 변수와 마찬가지로 블록 레벨 스코프를 가지며, 변수 호이스팅이 발생하지 않은 것처럼 동작한다.
- 재할당 금지
    - var, let과 달리 const 키워드는 재할당이 금지된다.
- 상수
    - **const 키워드로 선언한 변수에 원시 값을 할당한 경우 재할당이 안되기 때문에 변수 값을 변경할 수 없다**.
    (원시값 할당의 경우 원시 값은 불변 값이라 재할당이 아니면 변경이 안되기 때문) 
    따라서 상수를 표현하는데 사용된다.(상수는 재할당이 금지된 변수이기 때문)
- const 키워드와 객체
    - const 키워드로 선언된 변수에 **객체를 선언한 경우에는 값을 변경할 수 있다**. 객체는 재할당 없이도 직접 변경이 가능하기 때문이다.
    - **const  키워드는 재할당을 금지할 뿐 불변을 의미하지는 않기에 변경에 재할당이 필요한 원시값은 변경이 불가능하나 객체값은 가능하다.**

## 15.4 `var` vs `let` vs `const`

- ES6을 사용한다면 var 키워드는 사용하지 않는다.
- 재할당이 필요한 경우에 한정해 let 키워드를 사용한다. 이때 변수의 스코프는 최대한 좁게 만든다.
- 변경이 발생하지 않고 읽기 전용으로 사용하는 원시값과 객체에는 const 키워드를 사용하여 재할당을 금지하자.
